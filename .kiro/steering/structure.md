# Project Structure

## Root Layout
```
├── client/          # React frontend application
├── server/          # Express backend API
├── shared/          # Shared types and schemas
├── script/          # Build scripts
└── attached_assets/ # Static assets
```

## Client Structure (`client/`)
```
├── src/
│   ├── components/     # React components
│   │   ├── ui/        # shadcn/ui components (Radix primitives)
│   │   ├── Layout.tsx # Main layout wrapper
│   │   └── ProductCard.tsx
│   ├── pages/         # Route components (Home, Catalog, Cart, etc.)
│   ├── hooks/         # Custom React hooks
│   │   ├── use-cart.ts      # Zustand cart state
│   │   ├── use-products.ts  # TanStack Query hooks
│   │   └── use-orders.ts
│   ├── lib/           # Utilities
│   │   ├── queryClient.ts   # TanStack Query config
│   │   └── utils.ts         # cn() helper for class merging
│   ├── App.tsx        # Router setup
│   └── main.tsx       # Entry point
├── index.html         # HTML template
└── requirements.md    # Frontend notes
```

## Server Structure (`server/`)
```
├── index.ts    # Express app setup, middleware, logging
├── routes.ts   # API route handlers + database seeding
├── storage.ts  # Database operations (Drizzle queries)
├── static.ts   # Static file serving (production)
└── vite.ts     # Vite dev server integration (development)
```

## Shared Structure (`shared/`)
```
├── schema.ts   # Drizzle table definitions, Zod schemas, types
└── routes.ts   # API route definitions with Zod validation
```

## Key Conventions

### API Routes
- All API routes prefixed with `/api`
- Route definitions in `shared/routes.ts` with Zod input/output schemas
- Handlers in `server/routes.ts`

### Database Layer
- Schema definitions: `shared/schema.ts` (Drizzle tables + Zod schemas)
- Data access: `server/storage.ts` (all database queries)
- Migrations: `migrations/` (generated by drizzle-kit)

### Component Organization
- UI primitives: `client/src/components/ui/` (shadcn/ui, don't modify directly)
- Feature components: `client/src/components/` (custom components)
- Page components: `client/src/pages/` (route-level components)

### State Management
- Global cart state: Zustand (`use-cart.ts`)
- Server state: TanStack Query (`use-products.ts`, `use-orders.ts`)
- Form state: React Hook Form with Zod validation

### Styling
- Tailwind utility classes (prefer composition over custom CSS)
- Class merging: Use `cn()` helper from `lib/utils.ts`
- Component variants: Use `class-variance-authority` (cva)

### Type Safety
- Shared types exported from `shared/schema.ts`
- Strict TypeScript mode enabled
- Zod schemas for runtime validation
- Type inference from Drizzle schemas
